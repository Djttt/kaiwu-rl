# 清扫大作战 DIY DQN 网络与算法说明

本文档介绍了当前在 `agent_diy` 目录下实现的 DQN (Deep Q-Network) 强化学习算法。该实现旨在控制“小悟”机器人在复杂的环境中高效清扫污渍，同时学会躲避障碍、避免原地打转，并适时返回充电桩。

---

## 一、 算法目标

我们的终极目标是让机器人“聪明地扫地”。具体来说，它需要学会：
1.  **路径规划**：高效地遍历房间，不漏扫，不重复扫。
2.  **自我保护**：识别并躲避在场地中巡逻的“官方机器人”（NPC），避免碰撞导致任务失败。
3.  **电量管理**：在电量耗尽前，准确判断距离并及时返回充电桩。
4.  **绝不卡死**：在狭窄角落或复杂地形中，能迅速脱困，而不是原地撞墙。

---

## 二、 核心技术与网络架构

为了实现上述目标，我们采用了 **基于 Dueling 架构的 Double DQN** 算法，并结合了**多模态特征输入**。

### 1. 网络输入（机器人能“看”到什么？）
机器人的“眼睛”和“大脑”接收两部分信息：
*   **局部视野 (21x21 地图)**：机器人能看到以自己为中心，前后左右各延伸 10 格的地图。这个地图告诉它哪里是墙、哪里是污渍、哪里扫过了。
*   **全局状态**：包含当前步数、剩余电量、充电桩距离、最近的 NPC 距离与速度趋势、甚至是机器人走过的轨迹统计（有没有在绕圈子）。

### 2. 神经网络结构 (Dueling DQN)
大脑（神经网络）如何处理这些信息？我们使用了 `Dueling DQN` 架构。
*   **CNN 视觉处理**：局部视野（21x21地图）就像一张图片，我们用卷积神经网络（CNN）来提取图片中的空间特征（比如“前面是死胡同”或者“右边有一大片污渍”）。
*   **MLP 逻辑处理**：全局状态（电量、坐标等数字）通过全连接网络（MLP）进行处理。
*   **优势与价值双流 (Dueling)**：处理完的信息被分成两股水流：
    *   **价值流 (Value)**：评估“我现在所处的环境好不好？”（比如电量满、旁边全是污渍，就是好状态）。
    *   **优势流 (Advantage)**：评估“在当前状态下，我采取上下左右哪个动作更好？”。
    最终，这两股水流相加，输出每个方向的 **Q 值（预期总收益）**。

### 3. 算法机制 (Double DQN)
传统的 DQN 容易“盲目乐观”，高估某个动作的回报。
*   **Double DQN**：我们准备了两个网络（在线网络和目标网络）。在线网络负责“选择”看起来最好的动作，而目标网络负责冷静地“评估”这个动作到底值多少分。这有效避免了机器人因为一时的运气好而产生错误的路径偏好。

## 三、 特色设计：如何让机器人更聪明？

### 1. 合法动作掩码 (Legal Action Masking) —— 告别“撞墙”
在迷宫中，有些方向是墙壁，根本走不通。
*   **动作掩码**：环境会告诉机器人当前哪几个方向是畅通的。在探索和预测时，我们**强制忽略所有非法的方向**。这样机器人从第一步开始就不会浪费时间去学“撞墙会痛”，极大地提高了学习效率。

### 2. 硬脱困机制 (Hard Unstuck) —— 告别“原地打转”
即便有了掩码，机器人有时也会因为某些奇怪的参数而在两个格子之间反复横跳。
*   **强制封禁**：我们加入了一个“状态机”。如果监控到机器人连续 4 步都在同一个坐标，系统就会说：“你上一步选的动作明显不对，接下来两步，你不准再选这个动作！”这迫使它尝试新的方向，迅速跳出死胡同。

### 3. 丰富的奖励机制 (Reward Shaping) —— 胡萝卜加大棒
机器人是通过“得分”来学习的。我们设计了非常细腻的奖励系统：
*   **清扫与探索**：扫到新污渍、走到没去过的地方，给大奖。
*   **生存与躲避**：如果发现 NPC 正在高速靠近自己，或者自己电量极低却离充电桩很远，提前给出惩罚（扣分），让它学会未雨绸缪。
*   **惩罚兜圈子**：如果它在原地绕圈子且没有扫到新东西，会受到严厉的扣分。

### 4. 经验回放池 (Replay Buffer)
机器人每天的经历（看到什么、怎么走、得了多少分）都会存入一个“记忆池”。在训练时，它会随机从记忆池中抽取 64 个片段来复习。这种机制让机器人能反复从历史经验中学习，数据利用率极高。

---

## 四、 总结

这套 DQN 算法并不只是一个套公式的数学模型，而是一个结合了**游戏机制先验知识**的实用系统。通过“合法动作过滤”和“强制脱困”保障下限（不犯低级错误），通过“多维度特征”和“精细奖励”拔高上限，最终训练出一个高效、聪明的清扫机器人。
